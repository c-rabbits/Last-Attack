# Windows 방화벽에서 8080 포트 허용 방법

## 방법 1: 명령어 (관리자 권한 필요)

1. **PowerShell 또는 CMD를 관리자로 실행**
   - 시작 메뉴에서 "PowerShell" 또는 "명령 프롬프트" 검색
   - **우클릭** → **"관리자 권한으로 실행"** 선택

2. 아래 명령어를 **그대로 복사해서** 붙여 넣고 Enter

```bat
netsh advfirewall firewall add rule name="Last Attack Dev Server" dir=in action=allow protocol=TCP localport=8080
```

3. `확인` 이라고 나오면 성공입니다.

---

## 방법 2: 설정 화면에서 수동으로 하기

1. **Windows 방화벽 고급 보안** 열기
   - `Win + R` 누르고 `wf.msc` 입력 후 Enter  
   - 또는: **제어판** → **Windows Defender 방화벽** → 왼쪽 **"고급 설정"**

2. **인바운드 규칙** 선택
   - 왼쪽에서 **"인바운드 규칙"** 클릭

3. **새 규칙 만들기**
   - 오른쪽 **"새 규칙..."** 클릭

4. 규칙 마법사에서 순서대로:
   - **규칙 종류**: ○ **포트** → 다음
   - **프로토콜**: ○ **TCP**, **특정 로컬 포트**: `8080` 입력 → 다음
   - **작업**: ○ **연결 허용** → 다음
   - **프로필**: 도메인/개인/공용 **전부 체크** (또는 "개인"만 해도 됨) → 다음
   - **이름**: `Last Attack Dev Server` (아무 이름이나 가능) → 마침

5. 목록에 새 규칙이 생겼는지 확인한 뒤 창 닫기.

---

## 확인

- 모바일과 PC를 **같은 Wi‑Fi**에 연결한 뒤
- 휴대폰 브라우저에서 **http://(PC IP):8080** 으로 접속해 보세요.
- PC IP는 터미널에서 `ipconfig` 로 확인할 수 있습니다 (IPv4 주소).

## 규칙 삭제 (나중에 서버 안 쓸 때)

관리자 PowerShell/CMD에서:

```bat
netsh advfirewall firewall delete rule name="Last Attack Dev Server"
```
